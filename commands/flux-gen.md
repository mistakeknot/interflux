# Flux-Gen — Review Agent Generator

Generate `fd-*` review agents in `.claude/agents/` using LLM-designed specs tailored to the task at hand.

## Step 0: Detect Mode

Parse `$ARGUMENTS`:

- If `$ARGUMENTS` starts with `--from-specs` → **Specs mode**: skip Step 1 (LLM design), read the specs file path, and go directly to Step 4 (generate from saved specs)
- Otherwise → **Prompt mode** (Step 1) — `$ARGUMENTS` is treated as the task description

If `$ARGUMENTS` is empty (flux-gen invoked without arguments), build a default task description:

1. Read project root for README.md, build files (go.mod, package.json, Cargo.toml, etc.), and CLAUDE.md
2. Derive: `/flux-gen "Review of {PROJECT_ROOT basename}: {1-line project description}"`

---

## Step 1: Design Agent Specs

Launch a **Sonnet** subagent (Task tool, `model: sonnet`) to design 3-5 focused review agents for the task. The subagent outputs a JSON array of objects, each with: `name` (fd-*), `focus`, `persona`, `decision_lens`, `review_areas` (4-6 bullets), `severity_examples` (2-3 P0/P1 scenarios with condition), `success_hints`, `task_context`, `anti_overlap`. Design rules: distinct non-overlapping focus, descriptive names (fd-domain-concern), specific actionable review areas, concrete severity examples (not category restatements). Severity reference: P0=blocks/data loss, P1=blocks quality gate, P2=degrades quality, P3=improvements.

Parse the JSON. If parsing fails, offer retry.

## Step 2: Preview Specs

Display the designed agents to the user:

```
Designed {N} task-specific agents:

  1. fd-{name1}: {focus}
  2. fd-{name2}: {focus}
  3. fd-{name3}: {focus}
  ...
```

## Step 3: Confirm

Use **AskUserQuestion** to confirm:
- Option 1: "Generate {N} agents (Recommended)" → proceed
- Option 2: "Regenerate specs (different agents)" → go back to Step 1
- Option 3: "Cancel"

## Step 4: Generate

1. **Save specs** to `{PROJECT_ROOT}/.claude/flux-gen-specs/{slug}.json` for future regeneration without re-running the LLM design step. Derive `{slug}` from the task prompt (e.g., `mcp-agent-mail-research`, `auth-refactor-review`). Use the Write tool.

2. **Validate specs** before writing: verify the JSON is a non-empty array where each object has a `name` field starting with `fd-`. If validation fails, show the raw subagent output and offer to retry Step 1.

3. **Write specs** to a temp file using the Write tool (NOT heredoc/echo in Bash) and run the generation script:

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/scripts/generate-agents.py {PROJECT_ROOT} --from-specs /tmp/flux-gen-specs.json --mode=skip-existing --json --verbose
```

Parse the JSON report. If `generated` is empty and `errors` contains a warning, display it prominently — this means the specs were malformed or all agents already exist.

## Step 5: Report

Report generated agents (name, focus, status). Note specs path and regeneration command. Agents are available to flux-drive immediately (Project Agents, +1 bonus). Using as direct subagent_types requires session restart.

## Notes

- Generated agents use `subagent_type: general-purpose` in flux-drive (content pasted as system prompt)
- v5 agents have severity calibration; v4 have persona/decision lens/anti-overlap. Older versions auto-regenerated by `--mode=regenerate-stale`
- Specs saved to `.claude/flux-gen-specs/{slug}.json` (~25k tokens saved per regeneration)
- Tagged `generated_by: flux-gen-prompt` in frontmatter
