# Track B — Newsroom Wire Desk

**source_domain:** Newsroom wire-desk / editorial aggregation (a veteran wire editor merging many reporters' and agencies' filings into one authoritative story under deadline).
**distance_rationale:** Same abstraction level as interflux synthesis — reconcile many independent sources into one trustworthy output — but in journalism, not software.
**expected_isomorphisms:** source attribution & provenance, corroboration vs single-source, conflict reconciliation, deadline/length budgets, spiking weak filings, bias/echo detection, byline accountability.

---

- **[P1] Cross-track "convergence" is treated as corroboration without checking source independence** — `skills/flux-review-engine/phases/track-synthesis.md:3,36-43` — Editorial principle: two outlets running the same wire copy are *one* source, not two; a wire editor never counts AP and a paper that reprinted AP as "independent confirmation." track-synthesis.md calls convergent findings "the highest-confidence signal" *because* they came from "independent reasoning paths," and ranks purely by `convergence score: how many independent tracks found it (2/2, 2/3, 3/4)`. But when all track agents share a base model, agreement is correlated error (echo chamber), not corroboration — the synthesis prompt has no field for, and gives the synthesizer no instruction to discount, shared-model agreement. The single-run synthesizer *does* know this (see hearsay/cross-family logic below), but that knowledge is dropped at the cross-track tier. Concrete fix: pass each track agent's `model_family` into the track-synthesis prompt and instruct it to down-weight convergence among same-family tracks (mirror the 1.5x cross-family multiplier from `docs/spec/core/synthesis.md:142-162`); label same-model convergence "correlated, treat as ~1 source."

- **[P1] Research agents never emit the provenance fields synthesis validates for — a "byline" mismatch that silently degrades attribution** — `agents/research/best-practices-researcher.md:58-62`, `agents/research/repo-research-analyst.md:53-74`, vs `skills/flux-engine/phases/synthesize.md:17-23` — Editorial principle: every filing must arrive in house style with a byline, sourcing block, and confidence note, or the desk can't attribute it. Synthesize Step 3.1 validates that each research agent's output contains `### Sources`, `### Findings`, `### Confidence`, and `### Gaps` plus a `<!-- flux-research:complete -->` sentinel. But none of the five `agents/research/*.md` definitions instruct the agent to emit those sections or that sentinel — best-practices uses "Must Have/Recommended/Optional" with inline "Source Attribution" prose; repo-research-analyst uses "Repository Research Summary"; framework-docs uses its own 7-point format. Every research agent will therefore be classified Error/Missing or fall to prose fallback, losing structured Sources and Confidence. Concrete fix: add the canonical `### Sources / ### Findings / ### Confidence / ### Gaps` + sentinel contract to each research agent's Output Format, or relax Step 3.1 to a documented prose-extraction path; do not let the validator and the reporters disagree on house style.

- **[P1] Source provenance is captured per finding but the standard report template has no column to surface it — attribution dies at the last mile** — `skills/flux-engine/phases/synthesize.md:200-209` vs `375-425` — Editorial principle: the byline runs in the printed paper, not just the editor's notes. findings.json carries `provider`, `model_family`, `model_id` per finding (synthesize.md:199-208; richer per-agent `provenance` map in `docs/spec/core/synthesis.md:229-233`), and the spec flags `verification_recommended` for single-source non-top-tier P0/P1 (synthesis.md:164-166). But the user-facing Step 3.5 report (synthesize.md:379-438) has no provenance/model column in the Verdict or Cost tables and never renders `verification_recommended`. A reader cannot tell that a scary P0 came solely from a cheap model. Concrete fix: add a "Source" column (model_family + convergence) to the Verdict Summary table and render a "Verify independently" badge on every `verification_recommended` finding, matching the spec's own example summary at synthesis.md (spec) lines 310-315.

- **[P1] Conflicting recommendations are preserved, but the conflict is never adjudicated or even guaranteed to print** — `skills/flux-engine/phases/synthesize.md:117-118`, `docs/spec/core/synthesis.md:87-92,354-356` — Editorial principle: when two reporters file contradictory facts, the wire editor either reconciles them, attributes both with a "sources differ" standfirst the reader can't miss, or spikes the weaker one — never silently prints both buried mid-story. Rule 5 says "preserve both with attribution... Do NOT pick a winner — the user decides," and Rule 4 takes the *highest* severity. That is defensible for severity, but for substantive recommendation conflicts there is no requirement to escalate visibility: the Conflicts section (synthesize.md:404-405) defaults to "No conflicts detected" and only severity conflicts are spec'd to populate it (synthesis.md:354-356 shows "No severity conflicts detected" even when recommendations diverge). Two agents giving opposite fixes can both land in the body with the user never told they conflict. Concrete fix: route Rule-5 recommendation conflicts into the Conflicts section as a first-class "sources differ" entry, and require the synthesizer to add a one-line reconciliation note (which fix dominates and why, or "genuinely undecidable — user must choose").

- **[P2] "Spiking" only exists as failure/budget pruning — there is no quality spike for a completed-but-weak filing** — `skills/flux-engine/phases/synthesize.md:38-43` (Foreign/Error/Malformed), `docs/spec/extensions/agent-dropout.md`, `config/flux-drive/reaction.yaml:budget cap` — Editorial principle: a desk spikes a filing that is thin, unsourced, or off-beat even when it arrived on time and in format — completeness is not credibility. interflux can drop agents *before* they run (AgentDropout, budget cap) and discard *broken* outputs (Foreign/Error/Malformed), but a valid output full of low-confidence, evidence-free P2s is fully ingested and counted toward M. There is no synthesis-time "spike this filing" gate on quality. Concrete fix: at synthesis, compute a per-agent credibility score (share of findings with file:line evidence anchors; the compounding agent already gates on "evidence anchors" at synthesize.md:550-556) and either down-rank or quarantine a completed agent whose findings are overwhelmingly anchorless, reporting it as "spiked: low evidence" rather than counting it as a corroborating voice.

- **[P2] Hearsay/independence discounting is wired into the single-run reaction round but not carried into final convergence display or cross-track synthesis** — `config/flux-drive/reaction.yaml` (`hearsay_detection`, `convergence_weight_hearsay: 0.0`), `skills/flux-engine/phases/synthesize.md:348-356` vs `skills/flux-review-engine/phases/track-synthesis.md:36-43` — Editorial principle: a confirmation that merely cites another reporter ("as the AP reported") adds zero corroboration; the desk must mark it as derivative. interflux gets this exactly right *inside one run*: hearsay confirmations count 0.0, independent 1.0, and Step 3.4d emits `hearsay_count` / `independent_count` / `contested_count`. But (a) the user-facing report template (synthesize.md:379-438) never surfaces these counts, so the reader can't see how much of a finding's "convergence" was independent vs derivative; and (b) the cross-track synthesizer recomputes convergence by naive head-count with no hearsay notion at all. Concrete fix: print independent-vs-total alongside each finding's convergence ("3/4 agents, 2 independent"), and feed the same independent/hearsay distinction into track-synthesis convergence scoring.

- **[P2] No deadline/length budget on the synthesized story — the desk has a word count, the synthesizer does not** — `skills/flux-engine/phases/synthesize.md:379-438`, `skills/flux-review-engine/phases/track-synthesis.md:81-100` — Editorial principle: the front page has fixed inches; the editor must cut to the most important corroborated facts, not print every filing. The token *budget* governs input dispatch (cost_report, deferral, dropout), but the synthesis *output* has no length ceiling and no "above-the-fold" cut: Step 3.5 enumerates all P0/P1, all improvements, full heat map, full cost table; track-synthesis emits up to six themed sections (Critical / Cross-Track / Track A-D / Assessment) with no cap. For a many-agent or many-track run this buries the lede. Concrete fix: add an output budget — e.g., "lede: top N corroborated findings ranked by independent convergence, then the long tail collapsed under a `<details>` — so the highest-confidence, multiply-sourced findings always lead.

- **[P3] Foreign-file detection guards against cross-run contamination but there is no de-duplication of the same source quoted twice within a run** — `skills/flux-engine/phases/synthesize.md:27-32` — Editorial principle: the desk also watches for the same agency story arriving via two feeds under different slugs (double-counting one source). The quire-mark/run-uuid check (synthesize.md:27-32) correctly rejects stale files from prior runs, and Rule 1 merges same file:line+same issue. But if the *same* underlying external source (e.g., one Exa/Context7 doc) is cited independently by two research agents, nothing collapses it to one source — both count toward "Sources" and toward apparent corroboration. Concrete fix: in research synthesis, dedup the Sources list by URL/identifier and note when two agents leaned on the same source, so a single document is not mistaken for two-source corroboration.
