# Track A — Quality & Style Review (fd-quality)

## Overall Assessment

interflux is a mature, well-engineered plugin whose *executable* code quality is consistently high: the TypeScript MCP server (`mcp-servers/openrouter-dispatch/index.ts`) uses atomic temp-file writes, exclusive-lock acquisition with stale-lock breaking, EX_CONFIG exit codes, and careful narrow typing; the security-critical `scripts/sanitize_untrusted.py` is conservative, well-documented, and idiomatic; shell scripts that need strict mode use `set -euo pipefail`, and those that deliberately do not (`cost_capture.sh`, `teams_detect.sh`) make a defensible choice because they produce structured JSON and own their own error paths. The dominant quality problem is **not the code — it is drift between documentation, validation scripts, and reality.** The skills directory was renamed (`flux-drive/` → `flux-engine/` + `flux-review-engine/`) and a roster header was renamed (`(clavain)` → `(interflux)`), but CLAUDE.md, AGENTS.md, and at least one validation script were not updated in lockstep, leaving a validation script that always fails and several authoritative docs that contradict the manifest. The 12 review agents have drifted in section structure, and the structural test suite is shallow relative to the risk surface (security-critical Python modules are entirely untested; tests assert counts but not the cross-field invariants that the docs themselves claim matter). Order of findings: structural/validation breakage first, then doc-vs-reality contradictions, then agent consistency, then test coverage, then minor robustness.

---

- **[P1] `validate-roster.sh` is dead — points at a renamed skill path and a renamed header, always exits 1** — `scripts/validate-roster.sh:7,17,62` — The script hardcodes `SKILL_FILE="$PROJECT_ROOT/skills/flux-drive/SKILL.md"`, but the skill was reorganized into `skills/flux-engine/SKILL.md` + `skills/flux-review-engine/SKILL.md`. Running it yields `Mismatch: missing skill file ... skills/flux-drive/SKILL.md` and `exit=1` unconditionally. Even if the path were fixed, line 17 greps for the section header `### Plugin Agents (clavain)` while the roster (`skills/flux-engine/references/agent-roster.md:21`) now reads `### Plugin Agents (interflux)`, and the roster table lives in `references/agent-roster.md`, not in `SKILL.md` at all. A validation gate that can never pass is worse than no gate — it trains maintainers to ignore red. **Fix:** repoint `SKILL_FILE` to `skills/flux-engine/references/agent-roster.md`, update the awk matcher to `^### Plugin Agents \(interflux\)`, and add this script to CI so the path can't silently rot again.

- **[P1] Skill directory rename not propagated to AGENTS.md / scripts/README.md** — `AGENTS.md:108-112,216,229`, `scripts/README.md:174` — AGENTS.md documents the skill layout as `skills/flux-drive/` with `phases/` and `references/` subdirectories, and the phase/reference tables (lines 216-237) all reference `skills/flux-drive/phases/...`. The real layout is `skills/flux-engine/` and `skills/flux-review-engine/`. `scripts/README.md:174` likewise cites `skills/flux-drive/phases/shared-contracts.md`. Anyone following AGENTS.md to locate the orchestration protocol hits a dead path. **Fix:** global rename `skills/flux-drive/` → `skills/flux-engine/` across AGENTS.md and scripts/README.md (and audit the architecture topic guides under `agents/*.md` for the same).

- **[P1] CLAUDE.md self-contradicts on MCP server count; AGENTS.md undercounts** — `CLAUDE.md:7,24`, `AGENTS.md:17` — CLAUDE.md's overview says "2 MCP servers (exa, openrouter-dispatch)", but its own Quick-Commands validation one-liner (line 24) asserts the manifest contains exactly `['exa']`. The real `.claude-plugin/plugin.json` defines **both** `exa` and `openrouter-dispatch`, so the documented check is wrong and would fail if anyone ran it. AGENTS.md:17 compounds this by listing "1 MCP server (exa)" and "4 commands" (actual: 7). The README (`README.md`) is the only doc that describes reality. **Fix:** update CLAUDE.md line 24 to expect `['exa', 'openrouter-dispatch']`, fix AGENTS.md's Quick Reference component line, and consider deriving these counts in CI rather than hand-maintaining them in three places.

- **[P1] `agent-roles.yaml` and AGENTS.md disagree on fd-systems tier** — `AGENTS.md:147` vs `config/flux-drive/agent-roles.yaml:33` and frontmatter `agents/review/fd-systems.md:4` — AGENTS.md's cognitive-agent table lists fd-systems as "planner (opus tier)", but the frontmatter ships `model: haiku` and `agent-roles.yaml:33` explicitly annotates `fd-systems frontmatter=haiku (intentional, per git history)`. The config is the source of truth; the doc is stale and overstates the model tier by two levels, which misleads anyone reasoning about cost/quality. **Fix:** change the AGENTS.md fd-systems row to reflect the haiku/checker reality (or whatever role-tier actually maps), and add a structural test asserting frontmatter `model:` matches the tier documented in `agent-roles.yaml` (see test-coverage finding below).

- **[P2] Review-agent section structure has drifted — 3 of 7 technical agents lack the shared "What NOT to Flag" / "Focus Rules" sections** — `agents/review/fd-correctness.md:9-54`, `fd-architecture.md:9-47`, `fd-user-product.md:9-50` — The persona contract these agents are written to (visible in fd-quality, fd-safety, and all 5 cognitive agents) has five-ish sections including **What NOT to Flag** and **Focus Rules** — explicit scope-discipline guards that prevent reviewer over-reporting. fd-architecture and fd-user-product have **Focus Rules** but no **What NOT to Flag**; fd-correctness has **neither** (it uses bespoke `## Failure Narrative Method` / `## Prioritization` headers instead). fd-correctness is a safety-floor agent that runs on most reviews, so missing scope guards there has the highest blast radius. This is inconsistency, not a bug, but it means agents give noticeably different self-restraint signals. **Fix:** add a `## What NOT to Flag` section to fd-architecture, fd-correctness, fd-user-product, and either rename fd-correctness's `## Prioritization` to the canonical `## Focus Rules` or add one, so all 12 share the documented skeleton.

- **[P2] Security-critical Python modules have zero unit tests** — `scripts/sanitize_untrusted.py` (no `tests/test_sanitize*`), also `spec_types.py`, `lib_registry.py`, `_verification.py` untested — `sanitize_untrusted.py` is the B3 prompt-injection reference implementation guarding **four** untrusted channels (peer findings, LLM-authored agent specs, knowledge context, domain overlays) per its own docstring (lines 4-14), with intricate regexes (anchored override patterns, a multi-alternative base64 heuristic, NFKC + Cf-category stripping). Regexes this subtle are exactly where silent regressions hide, yet there is no test file. The docstring even promises "Epic C3 extends it with hypothesis fuzz tests" — the tests appear never to have landed. `lib_registry.py` performs atomic registry mutation under flock for five callers and is also untested. **Fix:** add `tests/test_sanitize_untrusted.py` with positive cases (each bypass class the regexes target: fullwidth, RLO, fenced code, base64, `NEW INSTRUCTIONS:`) and negative cases (legitimate prose like "never ignore the schema" must survive). Prioritize this over breadth elsewhere — it is the highest risk-to-coverage gap in the repo.

- **[P2] Structural tests assert counts but not the cross-field invariants the docs rely on** — `tests/structural/test_agents.py:23,41-48` — `test_agent_count` checks `== 17` and `test_all_fd_agents_present` lists only **8** of the 12 review agents (the docstring literally says "All 8 fd-* agents exist", a stale relic from before the 5 cognitive agents were added — 4 review agents are unverified by name). No test asserts: (a) frontmatter `name:` equals the filename stem, (b) `model:` is one of the allowed tiers, (c) `model:` agrees with `agent-roles.yaml`, or (d) the shared section skeleton from the finding above. These are precisely the invariants that drifted (fd-systems tier, missing sections). **Fix:** extend the existing parametrized `test_agent_is_nonempty` pattern to also assert name==stem and a valid `model:` enum, add a roles-drift test (you already have `verify_frontmatter.py --strict` — wire it into pytest), and update the "8" docstring + expected list to the full 12.

- **[P2] `cost_capture.sh` interpolates command output directly into JSON, risking malformed output** — `scripts/cost_capture.sh:97,101,119` — `rows` and `cost` come from `jq -r '... // 0'` over `interstat` output, then are spliced **unquoted** into hand-built JSON: `per_session_json+="{\"session_id\":\"$sid\",\"rows\":$rows,\"cost_usd\":$cost}"` and `"total_usd": $total`. If interstat ever returns a non-numeric token (e.g. `null`, `1,234`, an error word that slips past the `// 0`), the emitted document is invalid JSON and the caller's `jq` parse fails downstream rather than at the source. The script otherwise validates `session_id` carefully (line 70), so the numeric fields are the weak link. **Fix:** coerce with `jq` for the whole output object instead of string concatenation (build the array with `jq -n --argjson`), or at minimum validate `[[ "$rows" =~ ^[0-9]+$ ]]` / `[[ "$cost" =~ ^[0-9]+(\.[0-9]+)?$ ]]` before splicing.

- **[P3] Hardcoded machine-specific absolute paths in a shipped script and dev docs** — `scripts/cost_capture.sh:28`, `AGENTS.md:349`, `CLAUDE.md:17` — `cost_capture.sh` defaults `INTERSTAT_SCRIPT` to `/home/mk/projects/Sylveste/interverse/interstat/scripts/cost-query.sh` — a path that only exists on one developer's machine; on any other host the script silently returns `status: incomplete` (it does fall back gracefully, which is why this is P3 not P1, but the default is effectively never valid for other users). AGENTS.md:349 tells contributors to `cd /home/mk/projects/Demarch/interverse/interflux` and CLAUDE.md:17 references `/root/projects/Interverse/plugins/interflux` — three different absolute roots across the docs for the same repo. **Fix:** resolve interstat via `command -v` / `$CLAUDE_PLUGIN_ROOT`-relative discovery or an env var with a portable fallback; replace absolute paths in docs with repo-relative ones (`uv run pytest tests/ -q` from repo root).

- **[P3] `discover-models.sh` uses `eval` for dynamic variable lookup where a nameref is cleaner and safer** — `scripts/discover-models.sh:83` — `eval task="\$${tier^^}_TASK"` reconstructs a variable name from the loop variable. The values are controlled (`checker`/`analytical`/`judgment`), so this is not currently injectable, but `eval` for indirect expansion is fragile and flags on any security lint. **Fix:** use a bash nameref — `local -n task_ref="${tier^^}_TASK"; task="${task_ref:-}"` — or an associative array keyed by tier. Eliminates `eval` with no behavior change.

- **[P3] Two hooks run `set -uo pipefail` without `-e`, relying on a broad `trap ... ERR` — easy to misread as strict mode** — `hooks/session-start.sh:1-3`, `hooks/write-capabilities.sh:1` — `session-start.sh` uses `set -uo pipefail` + `trap 'exit 0' ERR`, deliberately swallowing all errors so a hook failure never blocks a session (a reasonable hook-safety choice). But `write-capabilities.sh` uses `set -uo pipefail` with **no** ERR trap and no `-e`, so a mid-script failure neither aborts nor is caught — partial capability files could be written silently. The two hooks make different no-fail bargains without a comment explaining the intent in the second. **Fix:** add a one-line comment in each hook documenting the chosen failure posture, and in `write-capabilities.sh` either add the same `trap 'exit 0' ERR` guard or guard the write so a partial file is never left behind.

- **[P3] `launch-qmd.sh` is documented as vestigial but still present and referenced** — `scripts/launch-qmd.sh`, `CLAUDE.md:245`, `AGENTS.md:245` — The script is now a two-line stub echoing "qmd MCP server has moved to the interknow plugin", and the docs correctly call it "vestigial". Keeping dead launchers around invites confusion (someone may wire it into a config). Low severity because it's clearly labeled. **Fix:** if nothing references it in `plugin.json` (it doesn't), delete it and remove the AGENTS.md/CLAUDE.md callouts, or move it to an `archive/` note. Same applies to auditing whether `scripts/README.md`'s 40-script table still matches the directory after the FluxBench additions.

### Notes on what is genuinely good (not flagged)

- `mcp-servers/openrouter-dispatch/index.ts` — exemplary: atomic `writeFile→rename`, `O_CREAT|O_EXCL` locking with stale-break, `mode: 0o600`/`0o700`, narrow `unknown`-typed catch with `code` extraction, and a deliberate `process.exit(78)` (EX_CONFIG) with an explanatory comment. No `any`. This is the quality bar the rest of the repo should match.
- `scripts/teams_detect.sh` and `scripts/cost_capture.sh` correctly choose `set -u` over `-euo pipefail` because they own structured JSON output and explicit exit codes — flagging them for "missing -e" would be wrong, so they are not flagged.
- Python type-hint discipline is strong where it matters: `verify_frontmatter.py`, `triage-stats.py`, and `sanitize_untrusted.py` annotate public functions and use `from __future__ import annotations` / `X | None`. Only `verify_frontmatter.py:117 normalize_model(m)` lacks annotations — too minor to list separately.
